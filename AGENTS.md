# AGENTS.md

## 仓库定位

这个仓库是智能体 Skill 文件的线上收集与维护中心。根目录下每个一级文件夹代表一个独立技能，文件夹名称必须与技能名称一致。

## 维护规则

- 新技能放在仓库根目录，不要统一塞进 `skills/` 子目录。
- 每个技能必须包含 `SKILL.md`。
- `SKILL.md` 的 YAML frontmatter 只保留 `name` 和 `description`。
- 技能名使用小写英文、数字和连字符。
- 详细参考资料放入 `references/`，可执行代码放入 `scripts/`，模板资产放入 `assets/`。
- 不要在技能文件夹内堆放与技能运行无关的说明文档。

## 提交前检查

- 确认新增或修改的技能可以被单独阅读和复用。
- 确认脚本依赖写入该技能文件夹内的依赖声明。
- 确认 README 的技能列表与根目录实际技能保持一致。
