# Feishu Workflow Routing

## Messages

- Draft before sending unless the user explicitly asks to send.
- Include recipient, purpose, context, requested action, and deadline.
- For sensitive topics, keep language neutral and avoid assigning blame.

## Tasks

- Extract owner, due date, priority, source context, and acceptance criteria.
- If a task is vague, convert it into one concrete next action.

## Calendar

- Confirm date, time zone, duration, attendees, agenda, and meeting room or video link.
- When comparing time slots, report conflicts and recommended windows.

## Documents

- Identify whether the user needs reading, editing, summarizing, commenting, or exporting.
- Preserve source structure when summarizing formal documents.

## Approvals

- Separate approval facts from recommendation.
- Report current node, waiting approver, deadline, and risk if delayed.
